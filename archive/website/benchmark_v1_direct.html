Key Insight: CLS++ achieves 99.6% on adversarial detection (Cat 5) vs GPT-4's 46.8% — a 2.1x improvement. The phase memory engine knows what it doesn't know because gas-phase items (low consolidation strength) are invisible to retrieval. When information was never stored, the engine correctly reports "not mentioned" instead of hallucinating.

Limitation: Categories 1-4 use word-level Jaccard retrieval (no embeddings yet). The published baselines use full context windows (128k+ tokens). CLS++ retrieves only 10 memories via keyword match. Embedding-based retrieval (planned) is expected to close this gap significantly.

Methodology

How this benchmark was conducted.

Dataset: LoCoMo-10 (ACL 2024) — 10 multi-session conversations spanning 100+ days each.

Pipeline:
1. Ingest: All dialog turns injected into a fresh PhaseMemoryEngine per conversation. Each turn creates a Fact with thermodynamic state variables (s, Σ, τ, F, H, L).
2. Query: For each QA pair, the engine's search() retrieves top-10 memories ranked by -F(item) × relevance. An LLM generates the answer from this augmented context.
3. Evaluate: Token-level F1 with stemming per LoCoMo specification. Category 5 uses binary "not mentioned" detection.

LLM Backend: OpenAI (GPT-4o-mini) with Claude/Gemini failover.
Retrieval: Word-level Jaccard similarity (no embedding model used yet).
Runtime: 75 minutes for 1,986 QA pairs on MacOS (single-threaded).
